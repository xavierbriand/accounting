Backlog-refiner agent + `/refine-backlog` command (issue #161, F8). A **propose-only**
`backlog-refiner` sub-agent runs six read-only analysis passes over the GitHub tracker —
aging/stale items, label integrity, umbrella/checkbox drift, superseded duplicates,
tripwire re-validation — and emits a **Backlog refinement report** ending in a tagged
proposed-actions table. The agent never mutates the tracker: `/refine-backlog` presents
the report, the user tags each action, and the **main session** executes only the approved
ones. `gh`-CLI-only (no MCP — removed in maint-20), bounded output, reviewer-spec house
style, airtight Never list. Acceptance was the first run: it independently surfaced all 9
items of the health-check reset-inventory fixture plus the § 8 numbering hole between R21
and R23 (fed to h10). Also **codified R26**: `.claude/agents`, `.claude/commands`, and skill specs are
harness → **Reduced** lane, not Light (Light narrows to docs/process/harness doc-only) —
h9 is the first story to self-select a lane, and it corrected the classification it was
routed by. The one-time tracker reset (executing the approved actions) runs post-merge,
user-gated. **Next: story-h10 (#162)** — single-source the rule system.
