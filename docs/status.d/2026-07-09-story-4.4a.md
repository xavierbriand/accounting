Ingest non-interactive commit semantics shipped (closes #181). `accounting ingest
--non-interactive`/`--json` — previously an accidental dry-run that never persisted —
now commits through the shared `commitBatch` path (snapshot → `saveBatch` →
`TransactionIngested` → cleanup) when no low-confidence row is pending, and keeps
exit 2 + needs-review JSON + no persistence when one is. JSON shapes intentionally
untouched. First half of the story-4.4 split driven by the Phase-1 `--json` coverage
audit and the LLM-agents-as-consumer interview reframe; second half (story-4.4b:
uniform envelope, convention normalization, JSON errors on stderr, contract doc)
inherits the audit + fork decisions recorded in docs/plans/story-4.4a.md. Deferred:
#213 (`--dry-run` preview flag), #215 (empty-batch phantom `TransactionIngested`),
#180 annotated (non-atomic audit-event recording now reachable from scripted callers).
